# Core Rules — Универсальные правила разработки

## 1. Общие принципы

- Всегда начинай с плана (PLAN → APPLY → VERIFY)
- Маленькие итерации, частые коммиты
- Diff-review перед большими изменениями
- Тесты после каждого изменения логики

## 2. Git-поток

- Все изменения через PR
- Не коммитить напрямую в main
- Коммиты: `cursor: <описание изменений>`
- После изменений: `git add -A → git commit → git push -u origin HEAD`

## 3. Безопасность

- Никогда не показывать `.env`, секреты, токены
- Использовать абстракции для конфигурации
- Не логировать чувствительные данные

## 4. Структура проекта

- Monorepo: `/backend`, `/frontend`
- Не создавать новые папки без необходимости
- Следовать существующей архитектуре

## 5. Тестирование

- Backend: тесты для измененной логики
- Frontend: `npm run lint`, `npm run build`
- Smoke tests после деплоя
